Chat API Documentation

A realtime chat backend with REST endpoints and WebSocket (SignalR) events. All responses follow a standard envelope format.

Overview

👤

Users

Profiles, presence

💬

Rooms

Groups, channels

✉️

Messages

Send, edit, delete

📁

Files

Upload, blob

Base URL

http://localhost:5000

All API routes are prefixed with /api. WebSocket hub is at /hubs/chat.

Response Format

Every REST response uses a standard envelope:

{ "success": true, "message": "Success", "data": { ... } or [ ... ], "total": 42 // only on list responses }

Error Handling

StatusMeaning
200OK — success
201Created — resource created
400Bad Request — validation failed
404Not Found — resource missing
409Conflict — duplicate resource
500Server Error — unexpected failure

GET /api/users

Returns all registered users sorted by display name.

Response 200
{ "success": true, "data": [ { "id":"user_001", "username":"alice", "displayName":"Alice Johnson", "isOnline":true } ], "total": 3 }

GET /api/users/{id}

Fetch a single user by their ID.

ParamTypeWhereDescription
idstringpathUser's MongoDB ObjectId or seeded ID required

POST /api/users

Create a new user. Usernames must be unique.

{ "username": "john_doe", // required, unique "displayName": "John Doe", "email": "john@example.com", "avatarUrl": "https://..." }

PATCH /api/users/{id}

Partially update a user profile. All fields are optional.

{ "displayName": "Johnny", "avatarUrl": "https://...", "isOnline": true }

POST /api/users/{id}/status

Set a user's online/offline status. Also broadcasts UserOnline / UserOffline via WebSocket.

{ "isOnline": true }

DELETE /api/users/{id}

Permanently removes a user record.

GET /api/rooms

Returns all rooms sorted by latest message activity.

GET /api/rooms/user/{userId}

Returns only the rooms that include the given user as a member.

GET /api/rooms/{id}

Fetch a single room with member IDs and last-message preview.

POST /api/rooms

Create a new room. type can be 0 (Direct), 1 (Group), or 2 (Channel).

{ "name": "engineering", "description":"Engineering team chat", "type": 1, // 0=Direct, 1=Group, 2=Channel "memberIds": ["user_001", "user_002"], "createdBy": "user_001" }

POST /api/rooms/{id}/members

Add a user to an existing room.

{ "userId": "user_003" }

DELETE /api/rooms/{id}/members/{userId}

Remove a specific user from a room.

GET /api/messages/room/{roomId}

Returns paginated message history, ordered oldest → newest.

Query ParamTypeDefaultDescription
limitint50Messages per page
skipint0Offset for pagination

POST /api/messages

Send a message. Automatically broadcasts ReceiveMessage to all room WebSocket subscribers and updates the room's last-message preview.

{ "roomId": "room_001", "senderId": "user_001", "content": "Hello world! 👋", "type": 0 // 0=Text, 1=Image, 2=File, 3=System }

PATCH /api/messages/{id}

Edit message content. Emits MessageEdited to room subscribers. Only the original sender can edit.

{ "content": "Updated message text" }

DELETE /api/messages/{id}

Soft-deletes a message (content replaced with [Message deleted]). Emits MessageDeleted to room.

POST /api/messages/{id}/read

Mark a message as read by a specific user.

{ "userId": "user_001" }

GET /api/messages/room/{roomId}/unread?userId={userId}

Returns the count of unread messages in a room for a user (excludes their own messages).

POST /api/files/upload

Upload a file using multipart/form-data. Max 20 MB. Optionally post directly to a room.

⚠️ Allowed types: .jpg, .jpeg, .png, .gif, .webp, .pdf, .txt, .zip, .mp4, .mp3
Form FieldTypeDescription
filefileThe file to upload required
roomIdstringIf provided, creates a message in this room
senderIdstringIf provided with roomId, sets message sender

DELETE /api/files/{fileName}

Delete a previously uploaded file by its stored filename (UUID + extension).

WebSocket — Connect

The WebSocket hub is powered by ASP.NET Core SignalR. Connect using the @microsoft/signalr JS client or any SignalR-compatible client.

Endpoint: /hubs/chat
Optional query: ?userId=user_001 — triggers online-status broadcast on connect/disconnect.

WebSocket Events Reference

Client → Server (invoke)

MethodArgumentsDescription
JoinRoomroomIdSubscribe to all messages in a room
LeaveRoomroomIdUnsubscribe from a room
TypingroomId, userId, displayNameBroadcast typing indicator
StopTypingroomId, userIdStop typing indicator
Ping—Test connection; server replies with Pong

Server → Client (on)

EventPayloadDescription
ReceiveMessage{ id, roomId, senderId, senderName, content, type, timestamp }New message in room
MessageEdited{ id, content }Message was edited
MessageDeleted{ id, roomId }Message was deleted
UserTyping{ roomId, userId, displayName }User is typing
UserStoppedTyping{ roomId, userId }User stopped typing
UserOnline{ userId, connectedAt }User connected
UserOffline{ userId, disconnectedAt }User disconnected
UserJoinedRoom{ roomId, connectionId, joinedAt }Someone joined a room
UserLeftRoom{ roomId, connectionId, leftAt }Someone left a room
Pong{ timestamp }Reply to Ping

WebSocket JS Example

// Install: npm install @microsoft/signalr import * as signalR from "@microsoft/signalr"; const conn = new signalR.HubConnectionBuilder() .withUrl("/hubs/chat?userId=user_001") .withAutomaticReconnect() .configureLogging(signalR.LogLevel.Information) .build(); // Listen for new messages conn.on("ReceiveMessage", (msg) => { console.log(`[${msg.senderName}] ${msg.content}`); }); // Listen for typing indicator conn.on("UserTyping", ({ displayName }) => { console.log(`${displayName} is typing…`); }); // Start and join a room await conn.start(); await conn.invoke("JoinRoom", "room_001"); // Send typing indicator await conn.invoke("Typing", "room_001", "user_001", "Alice"); // Send message via REST (triggers WS broadcast automatically) await fetch("/api/messages", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ roomId: "room_001", senderId: "user_001", content: "Hello! 👋" }) });
